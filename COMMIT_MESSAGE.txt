feat(rate-limiter): Add API keys management and rate-limiting system

Implement a complete rate-limiting and API key management system for Adsbot with support for Redis and SQLite backends, FastAPI middleware, and Flask decorators.

Features:
- API key role-based access control (admin/user) with secure .env storage
- Redis rate limiter with fixed-window counters (configurable)
- SQLite fallback implementation for environments without Redis
- Two rate-limiting policies: BLOCK (429 responses) and SLOWDOWN (exponential backoff)
- Exponential backoff algorithm: delay = min(10s, 0.5 * 1.5^(over-1))
- FastAPI ASGI middleware for automatic rate-limit enforcement
- Flask decorator wrapper for synchronous endpoints
- Admin keys exempt from all rate limits
- Test suite with unit tests for both Redis and SQLite implementations
- GitHub Actions CI workflow for automated testing
- Comprehensive documentation with curl examples

Configuration:
- Environment variables: ADMIN_API_KEYS, USER_API_KEYS, REDIS_URL
- Support for .env file via python-dotenv (development)
- Dynamic key reload without restart
